# GigOps Lite v0.3a Spec: Backend Smoke Test

## Purpose

Define the backend smoke-test milestone for GigOps Lite v0.3a so contributors and AI agents can verify core API behavior quickly and consistently.

This smoke test is part of the project harness and supports the Agentic Engineering workflow.

## Scope

- Validate that the local backend service is running and reachable.
- Validate core backend endpoints with requestId and expected response envelopes.
- Validate list endpoint pagination metadata.
- Validate idempotency behavior for POST /jobs.
- Validate task assignment simulation for a known task and agent.

## Non-Goals

- No backend feature expansion.
- No frontend integration.
- No deployment setup.
- No persistence layer integration.
- No cloud, auth, queue, or CI/CD wiring.

## Endpoints Validated

- GET /health
- GET /jobs?page=1&limit=10
- GET /tasks?page=1&limit=10
- GET /agents
- GET /audit
- POST /jobs
- POST /tasks/TASK-2002/assign

## Success Criteria

- Smoke test command completes with all checks passing.
- GET /health returns 200.
- GET /jobs and GET /tasks return 200, include requestId, and include pagination metadata.
- GET /agents and GET /audit return 200 and include requestId.
- First POST /jobs with Idempotency-Key returns 201.
- Replay POST /jobs with same Idempotency-Key and same payload returns 200 with same job id and idempotent replay marker.
- Replay POST /jobs with same Idempotency-Key and different payload returns conflict-style error (409).
- POST /tasks/TASK-2002/assign assigns task to AG-002.

## Manual Review Checklist

- Smoke test script exists and is readable.
- package script for smoke test exists and runs locally.
- README explains how to run backend and smoke test.
- No frontend files were changed for this milestone.
- No backend endpoint behavior was changed by the spec-only update.

## Mock and In-Memory Limitations

- Backend is local-only for this milestone.
- Data is in-memory and resets on process restart.
- No Firestore, GCP, Auth0, Pub/Sub, Docker, or CI/CD integration.
- Results validate API contract behavior, not production durability or scale.
