# GigOps Lite v0.3c Spec: Frontend API Client (Tasks Only)

## Purpose

Introduce a small, safe extension of the frontend API client so the tasks page can read from the standalone backend when configured, while preserving mock behavior by default.

## Scope

- Extend the existing frontend API client with tasks fetching.
- Connect only the tasks page to this client.
- Support runtime switching using NEXT_PUBLIC_API_BASE_URL.
- Keep deployed frontend safe by defaulting to the existing mock route behavior.

## Non-Goals

- No changes to backend endpoint behavior.
- No frontend integration for agents, audit, quality, payouts, or dashboard routes.
- No task assignment action wiring.
- No backend deployment, auth wiring, or infrastructure changes.
- No replacement or removal of existing mock API routes.

## Runtime Behavior

- If NEXT_PUBLIC_API_BASE_URL is set, task requests are sent to the standalone backend endpoint.
- If NEXT_PUBLIC_API_BASE_URL is not set, task requests are sent to the existing Next.js mock route.
- Task data is normalized into the existing frontend TaskItem shape for UI safety.

## Environment Variable Strategy

- Variable: NEXT_PUBLIC_API_BASE_URL
- Optional by design.
- Reuses the existing v0.3b environment switch.
- Example local value: http://localhost:4000

## Mock Fallback Behavior

- Default behavior remains mock-first.
- Without NEXT_PUBLIC_API_BASE_URL, the tasks page uses the frontend mock route (/api/tasks).
- This keeps the deployed frontend working without backend deployment.

## Manual Test Checklist

- With no NEXT_PUBLIC_API_BASE_URL set:
  - /tasks loads successfully.
  - Data is sourced from /api/tasks mock route.
- With NEXT_PUBLIC_API_BASE_URL set to local backend:
  - /tasks loads successfully.
  - Data is sourced from standalone backend /tasks endpoint.
- Verify assignment buttons remain UI-only and do not call backend.
- Verify no other pages were switched to the standalone backend.

## Success Criteria

- /tasks page can use the standalone backend when configured.
- /tasks page falls back to the mock route when not configured.
- No frontend crash in either mode.
- No change to deployed Vercel behavior when the env var is absent.
