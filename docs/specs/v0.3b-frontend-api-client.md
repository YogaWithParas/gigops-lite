# GigOps Lite v0.3b Spec: Frontend API Client (Jobs Only)

## Purpose

Introduce a small, safe frontend API client that can read jobs from the standalone backend when configured, while preserving existing mock behavior by default.

## Scope

- Add a frontend API client utility for jobs fetching.
- Connect only the jobs page to this client.
- Support runtime switching using NEXT_PUBLIC_API_BASE_URL.
- Keep deployed frontend safe by defaulting to existing mock route behavior.

## Non-Goals

- No changes to backend endpoint behavior.
- No frontend integration for tasks, agents, audit, quality, payouts, or dashboard routes.
- No backend deployment, cloud wiring, auth wiring, or infra changes.
- No replacement or removal of existing mock API routes.

## Runtime Behavior

- If NEXT_PUBLIC_API_BASE_URL is set, jobs requests are sent to the standalone backend endpoint.
- If NEXT_PUBLIC_API_BASE_URL is not set, jobs requests are sent to the existing Next.js mock route.
- Jobs data is normalized into the existing frontend JobBatch shape for UI safety.

## Environment Variable Strategy

- Variable: NEXT_PUBLIC_API_BASE_URL
- Optional by design.
- Not required for Vercel production runtime.
- Example local value: http://localhost:4000

## Mock Fallback Behavior

- Default behavior remains mock-first.
- Without NEXT_PUBLIC_API_BASE_URL, /jobs page uses frontend mock API route (/api/jobs).
- This keeps the deployed frontend working without backend deployment.

## Manual Test Checklist

- With no NEXT_PUBLIC_API_BASE_URL set:
  - /jobs loads successfully.
  - Data is sourced from /api/jobs mock route.
- With NEXT_PUBLIC_API_BASE_URL set to local backend:
  - /jobs loads successfully.
  - Data is sourced from standalone backend /jobs endpoint.
- Verify no other pages were switched to standalone backend.
- Verify existing mock routes remain present.

## Success Criteria

- /jobs page can use standalone backend when configured.
- /jobs page falls back to mock route when not configured.
- No frontend crash in either mode.
- No changes to deployed Vercel behavior when env var is absent.
