# GigOps Lite v0.1 Spec: Frontend + Mock API

## Purpose

Define the current, implemented scope of GigOps Lite v0.1 so contributors and AI agents can make safe, accurate changes.

## Implemented in v0.1

- Next.js App Router frontend
- TypeScript codebase
- Mock API route handlers under frontend/app/api
- Synthetic in-memory datasets for jobs, tasks, agents, QA, payouts, and audit events
- Local build and type verification
- Route smoke testing

## UI Routes

- /
- /jobs
- /tasks
- /agents
- /quality
- /payouts
- /audit

## API Routes (Mock)

- /api/jobs
- /api/tasks
- /api/agents
- /api/audit

Expected behavior:
- Return JSON payloads for mock operational views.
- No persistent storage side effects.
- No external service dependencies.

## Non-Goals for v0.1

- Production deployment wiring
- Persistent backend data model implementation
- Real authentication and authorization provider integration
- Real event bus integration
- Real payout processor integration

## Planned Beyond v0.1

- GCP Cloud Run services
- Firestore persistence
- Pub/Sub eventing
- Auth0-based auth and RBAC claims
- Production observability and alerting
- Audit-grade immutable logging
- Real payout processing

## Engineering Intent

This version is intentionally honest and modular:
- Frontend flows are concrete and demoable.
- Backend behavior is simulated for developer speed.
- Planned systems are documented but not yet integrated.
